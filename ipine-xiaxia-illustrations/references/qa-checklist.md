# 虾线橘猫 QA 质检清单

> 基于 `ian-xiaohei-illustrations` 的 `qa-checklist.md` 改写。
> 核心变化：把"小黑"替换为"虾线橘猫"，保留质检逻辑和迭代方法。

---

## 必过项

- [ ] 是 16:9 横版
- [ ] 背景是干净白底
- [ ] 有虾线橘猫
- [ ] 虾线橘猫承担核心动作，不只是装饰
- [ ] 背上的虾线清晰可见
- [ ] 没有复刻旧案例构图，而是为当前文章生成了新隐喻
- [ ] 画面怪诞、有创意、有意思
- [ ] 简洁清爽，主体不超过画面约 60%
- [ ] 一张图只讲一个核心结构
- [ ] 中文标注少、短、能读
- [ ] 橘色只用于虾线橘猫的橘色毛块、主路径或箭头
- [ ] 红色只用于重点、问题、提醒或结果
- [ ] 蓝色只用于补充说明、反馈或系统状态

---

## 失败信号

出现以下情况，重生成或局部编辑：

- [ ] 左上角有"常见坑 / Workflow / 系统架构图 / 路线图"等标题
- [ ] 虾线橘猫像吉祥物、表情包或可爱萌宠
- [ ] 虾线橘猫背上的虾线不可见或被弱化
- [ ] 画面像 PPT、课程课件、正式流程图
- [ ] 元素太多、箭头太多、节点太多
- [ ] 文字变成大段解释
- [ ] 背景有纸纹、阴影、渐变、米色、噪点
- [ ] 真实 UI 截图或科技感界面
- [ ] 中文错字严重或标注不可读
- [ ] 画面太死板，没有荒诞隐喻
- [ ] 和 `assets/examples/` 里的旧案例构图过于相似

---

## 迭代方法

| 问题 | 解决方法 |
|------|---------|
| 太普通 | 让虾线橘猫成为动作主体，加入一个奇怪但成立的隐喻 |
| 太复杂 | 删节点，只保留一个动作和 3-5 个短标注 |
| 太可爱 | 强调 deadpan、blank serious expression、not cute、not mascot |
| 虾线不清晰 | 调整构图让背部朝向观众，或让虾线橘猫转身 |
| 太 PPT | 去掉标题、边框、整齐网格和过多箭头，改成手绘场景 |
| 太像旧案例 | 保留核心意思，换掉主物件和虾线橘猫动作 |
| 文字错 | 优先局部编辑；错得多就重生成并减少标注数量 |

---

## 交付判断

高质量图应该让读者先觉得"有点怪"，然后 1 秒内看懂结构。

如果第一眼像教程页，而不是白纸上的怪诞产品草图，就不合格。

**额外判断**：如果去掉虾线橘猫，图的核心隐喻还能完全成立，说明虾线橘猫太装饰了；如果虾线橘猫背上的虾线不清晰，说明 IP 识别度不够。两者都不合格。
